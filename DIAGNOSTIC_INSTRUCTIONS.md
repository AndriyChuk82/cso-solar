# Інструкція з діагностики проблеми з ціною АКБ

## Проблема
Ціна для "DEYE BOS-G PRO 100AH" показує 0 або 100 замість 800.

## Додано діагностичне логування

### Білд: ✅ Успішний (2.07s)

## Як діагностувати

### Крок 1: Відкрийте консоль браузера
1. Натисніть `F12` або `Ctrl+Shift+I`
2. Перейдіть на вкладку **Console**

### Крок 2: Завантажте сторінку
1. Оновіть сторінку (`F5` або `Ctrl+R`)
2. Зачекайте завантаження товарів з Google Sheets

### Крок 3: Шукайте логи

#### Лог 1: Діагностика АКБ (🔍)
Шукайте повідомлення:
```
🔍 Діагностика АКБ: {
  model: "DEYE BOS-G PRO 100AH",
  col1_raw: "???",  // ЩО ТУТ?
  col2: "LiFePO4",
  col3: "100",
  col4: "12",
  priceStr: "???"
}
```

**Що перевірити:**
- `col1_raw` - що саме приходить з колонки B?
- `priceStr` - чи правильно присвоюється?

#### Лог 2: Помилка parsePrice (⚠️)
Якщо parsePrice не може розпарсити, побачите:
```
⚠️ parsePrice не зміг розпарсити: {
  original: "???",
  afterProcessing: "???",
  parsedValue: NaN або 0,
  isNaN: true/false
}
```

**Що перевірити:**
- `original` - що прийшло в parsePrice
- `afterProcessing` - що залишилось після обробки
- `parsedValue` - результат parseFloat()

#### Лог 3: Ціна = 0 (⚠️)
Якщо ціна стала 0, побачите:
```
⚠️ Ціна = 0 для товару: {
  sheet: "АКБ",
  model: "DEYE BOS-G PRO 100AH",
  priceStr: "???",
  col1: "???",
  col3: "100",
  col4: "12",
  col10: "",
  parsedPrice: { value: 0, currency: "USD" }
}
```

## Можливі сценарії

### Сценарій 1: col1 порожній
```javascript
col1_raw: ""  // ❌ Порожній!
```
**Причина:** Google Sheets не повертає значення для col1
**Рішення:** Перевірити формат комірки B в Google Sheets

### Сценарій 2: col1 містить формулу
```javascript
col1_raw: "=IF(...)"  // ❌ Формула!
```
**Причина:** getVal повертає формулу замість значення
**Рішення:** Використовувати `cell.v` замість `cell.f`

### Сценарій 3: col1 містить текст, але parsePrice не розпізнає
```javascript
col1_raw: "800 гот / 960 з ПДВ"  // ✅ Правильно
afterProcessing: ""  // ❌ Щось пішло не так
```
**Причина:** Regex не спрацював
**Рішення:** Покращити regex в parsePrice

### Сценарій 4: col1 містить число як текст
```javascript
col1_raw: "800"  // ✅ Правильно
parsedValue: 800  // ✅ Правильно
priceVal: 0  // ❌ Чому 0?
```
**Причина:** Проблема після parsePrice
**Рішення:** Перевірити логіку присвоєння priceVal

## Що робити далі

### Після перегляду логів, надішліть мені:

1. **Скріншот консолі** з усіма логами
2. **Текст логів** (скопіюйте з консолі):
   - 🔍 Діагностика АКБ
   - ⚠️ parsePrice не зміг розпарсити
   - ⚠️ Ціна = 0 для товару

3. **Інформацію з Google Sheets:**
   - Що точно написано в комірці B для цього товару?
   - Який формат комірки (текст, число, формула)?
   - Чи є там формула або просто текст?

## Швидка перевірка в Google Sheets

1. Відкрийте лист "АКБ"
2. Знайдіть рядок "DEYE BOS-G PRO 100AH"
3. Перевірте комірку B (ціна):
   - Що там написано?
   - Натисніть на комірку - чи є формула в рядку формул?
   - Який формат комірки? (Формат → Число → ?)

## Тимчасове рішення (якщо col1 порожній)

Якщо виявиться, що col1 завжди порожній для АКБ, можна спробувати:
```typescript
// Замість
priceStr = col1;

// Використати
priceStr = col1 || col10 || col5;  // Fallback на інші колонки
```

Але спочатку потрібно зрозуміти, ЧОМУ col1 порожній.

**Ключові файли для правки логіки:**
*   Модуль Складу: `warehouse/src/api/externalApi.js` (функція `parseGvizJson`).
