# Виправлення парсингу цін в api.ts - ФІНАЛЬНА ВЕРСІЯ

## Проблема
Для товару "Акумулятор DEYE BOS-G PRO 100AH" з ціною "800 гот / 960 з ПДВ" система витягувала ціну 100 (ємність з назви) замість 800 (реальна ціна).

## Корінь проблеми

### Проблема №1: Неправильний порядок операцій у `parsePrice`
Функція намагалася витягти число **після** видалення символів валюти, що не працювало для формату "800 гот / 960 з ПДВ".

### Проблема №2: Складна логіка вибору колонки для АКБ
Нова версія використовувала `isLikelyPrice()` з fallback на інші колонки. Якщо col1 був порожній, вона брала col3 (де "100Ah").

**У старій версії для АКБ просто:** `priceStr = col1;` - БЕЗ fallback!

## Виправлення

### 1. Виправлена функція `parsePrice` (рядки 173-201)

**КРИТИЧНА ЗМІНА:** Порядок операцій тепер **точно як у старій версії**:

```typescript
const parsePrice = (str: string) => {
  if (!str || str.trim() === '') return { value: 0, currency: 'USD' };
  let s = str.trim();
  let currency = 'USD';

  if (s.includes('€')) currency = 'EUR';
  if (s.includes('₴') || s.toLowerCase().includes('грн')) currency = 'UAH';

  // КРИТИЧНО: Спочатку обробляємо формат "800 гот / 960 з ПДВ"
  // Витягуємо перше число ДО видалення символів валюти
  if (s.toLowerCase().includes('гот') || s.includes('/')) {
    const match = s.match(/[\d\s,.]+/);
    if (match) {
      s = match[0];
    }
  }

  // Тепер видаляємо символи валюти та чистимо
  s = s.replace(/[$€₴]|грн/gi, '').trim();
  s = s.replace(/\s/g, '');
  s = s.replace(',', '.');

  const val = parseFloat(s);
  if (isNaN(val) || val <= 0) return { value: 0, currency };
  return { value: val, currency };
};
```

### 2. Спрощена логіка вибору колонки (рядки 264-274)

**ТОЧНО як у старій версії - БЕЗ складної логіки:**


```typescript
// Для АКБ: просто col1, як у старій версії

// У parsePrice: спочатку перевіряємо "гот" або "/"
if (s.toLowerCase().includes('гот') || s.includes('/')) {

```
